# 기획안

# 데이트 로그 프로젝트 기획서

## 1. 서비스 개요

커플이 함께 다녀온 데이트 경로를 지도 위에 기록하고,
장소, 사진, 감정, 추억을 저장할 수 있는 데이트 기록 앱

## 2. 핵심 사용자

- 연애 중인 커플
- 데이트 장소와 추억을 기록하고 싶은 사용자
- 사진첩보다 더 의미 있게 데이트를 정리하고 싶은 사용자

## 3. 핵심 기능

- 카카오 로그인
- 커플 코드 연결
- 지도 기반 데이트 경로 기록
- 방문 장소 저장
- 사진 업로드
- 날짜별 데이트 기록 조회

## 4. 확장 기능

- 커플 미니 캐릭터
- 캐릭터 방꾸미기
- 버킷리스트 위치 등록
- 오늘의 커플 질문
- 오늘의 데이트 소감
- 출석 기능
- 커플 랭킹
- 디데이 계산기
- 앱 테마 색상 변경

## 5. MVP 목표

가장 먼저 만들 버전은 “데이트 경로 기록 + 사진/장소 저장 + 이전 기록 조회”에 집중한다.

## 6. 개발 순서

1. **프로젝트 기획서 작성**
2. **MVP 기능 확정**
3. **화면 구성도 작성**
4. **DB 테이블 설계**
5. **API 명세 작성**
6. **프론트엔드 기본 화면 개발**
7. **백엔드 로그인 / 커플 연결 개발**
8. **지도 경로 기록 기능 개발**
9. **사진 업로드 / 장소 기록 개발**
10. **이전 기록 조회 기능 개발**

## 7. MVP 핵심 기능

1. **카카오 로그인**
2. **커플 코드 생성 / 입력 / 연결**
3. **데이트 기록 생성**
4. **지도에서 데이트 경로 그리기**
5. **경로 위 장소 저장**
6. **장소별 사진 / 메모 저장**
7. **이전 데이트 기록 리스트 보기**
8. **데이트 기록 상세 보기**

## 8. 기능 우선순위

### 1순위: 반드시 필요

| 기능 | 이유 |
| --- | --- |
| 로그인 | 사용자 구분 |
| 커플 연결 | 커플 앱의 핵심 |
| 지도 기록 | 서비스 핵심 |
| 장소/사진 저장 | 데이트 추억 기록 |
| 이전 기록 조회 | 기록 앱의 기본 기능 |

### 2순위: 있으면 좋음

| 기능 | 이유 |
| --- | --- |
| 버킷리스트 | 다음 데이트 계획으로 확장 가능 |
| 디데이 | 커플 앱 기본 감성 기능 |
| 오늘의 질문 | 재방문 유도 |
| 오늘의 소감 | 감정 기록 강화 |

### 3순위: 나중에

| 기능 | 이유 |
| --- | --- |
| 미니 캐릭터 | 개발량 큼 |
| 방꾸미기 | 콘텐츠/아이템 구조 필요 |
| 커플 랭킹 | 초기 유저 적으면 의미 약함 |
| 출석 점수화 | 보상 시스템 설계 필요 |
| 테마 색상 변경 | 핵심 기능 이후 추가 |
